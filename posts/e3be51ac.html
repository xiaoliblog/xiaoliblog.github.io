使用PicGo搭建GitHub图床

发表于 2023-03-03 | 更新于 2023-03-03
总字数: 453 | 阅读时长: 1分钟

GitHub图床

  • GitHub 图床是个不错的选择，利用 jsDelivr CDN 加速访问（jsDelivr 是一个免费开源的 CDN 解决方案），PicGo 工具一键上传，操作简单高效，GitHub 和 jsDelivr 都是大厂，不用担心跑路问题，不用担心速度和容量问题，而且完全免费，可以说是目前免费图床的最佳解决方案！

  • 须知

    • 需要绑定GitHub账户
    • 需要将 token 信息上传给后台

创建图片仓库

  • 登录/注册GitHub，新建一个仓库，填写好仓库名（我创建的是image仓库名，与图片不符），仓库描述，根据需求选择是否为仓库初始化一个README.md描述文件

  • 生成一个有 repo 权限的 GitHub Token

Typora配置

  • Typora配置好PicGo图床后，方便图片在线存储
  • 点击软件工具栏，文件 –> 偏好设置 –> 图像 –> 配置picgo

  • 可以看到127.0.0.1后面有个端口号36677，这个端口号一定要和picgo里面设置的端口号要一致！否则上传不成功！

其他图床

  • 我个人使用的是阿里云对象存储OSS搭建的图床，其上传速度更快，详情参考：https://blog.csdn.net/weixin_43571641/article/details/126534731
  • 如果不想下PicGO软件，也可以通过配置网上在线图床。推荐：https://picx.xpoet.cn/ ，只需创建GitHub参考和获取Token即可在线上传。可以生成并复制Jsdelive加速的CDN链接
