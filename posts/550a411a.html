15-Vue-Router路由管理器

发表于 2023-01-15 | 更新于 2023-03-03
总字数: 3.5k | 阅读时长: 16分钟

Vue-Router

  • Vue Router是 Vue.js官方的路由管理器。官方文档：https://router.vuejs.org/zh/
  • 路由器的功能：在数据通信时帮你选择通信的路线
  • 在Vue中的路由，能够帮助我们在一个Vue组件中实现其他组件的相互切换。也就是说，可以通过路由模块，再创建路由表，将制定的组件显示在路由视图中
  • 在SPA （single page web application）应用，即单页面应用中使用Vue Router做路径管理器。传统的页面应用，是用一些超链接来实现页面切换和跳转的。在Vue-router单页面应用中，则是路径之间的切换，也就是组件的切换。路由模块的本质就是建立起url和页面之间的映射关系，是一对key-value的关系，实现更新视图而不重新请求页面

基本使用

  • 下载安装Vue-Router模块
  • 在main.js中通过 Vue.use() 明确地安装路由功能
  • 实现两个组件之间进行切换

  • 首先创建两个组件
  • 在src下创建router/index.js，此文件用来管理所有路由。定义key-value，映射url和组件的关系
  • 然后在main.js中引入路由器并挂载
  • 在App.vue中使用<router-link to="/about">About</router-link>跳转，然后使用 <router-view></router-view>指定组件的呈现位置。其中active-class是router-link组件中的属性，用来做选中样式的切换；

注意要点

  • 路由组件不需要在App.vue中导入注册，不需要写组件标签。所有路由组件一般写在page文件夹下，一般组件存放在components下
  • 通过切换，隐藏了的组件其实默认被销毁了，使用的时候再去挂载

  • 每个组件都有自己的$route属性，里面存储着自己的路由信息
  • 整个应用只有一 个router， 可以通过组件的$router属性获取到

路由进阶

嵌套路由

  • 实现Home选项中嵌套两个组件，分别为News和Message，点击可以切换不同的信息

  • 首先添加News和Message两个组件
  • 然后在route/index.js中配置路由规则，在父路由配置新增children配置项，数组形式，路径不需要加/
  • 然后在Home.vue组件中使用，注意路径需要带上父路径

路由传参query

  • 实现在Home选项里的Message选项里选择，并传入编号id和标题title参数，并显示出来

  • 首先需要一个Detail组件获取数据并显示，query参数被挂载在每个组件实例的$route上
  • 然后需要配置路由信息，因为是Message里的项目，嵌套为子路由，出现三级嵌套
  • 然后在Message组件中准备数据，路由跳转并携带query参数，注意:to需要使用绑定指令绑定

命名路由

  • 命名路由就是给一个路由起一个唯一的名称，然后使用name配置项跳转调用这个名称即可，可以简化路由的跳转
  • 在路由配置项中新增name配置项即可指定名称
  • 使用的时候使用name指定，需要使用对象形式

params参数

  • 使用params参数传参，类似于Restful风格的传参。首先必须在路由配置中使用占位符声明接收params参数

  • 特别注意：路由携带params参数时，若使用to的对象写法，则不能使用path配置项，必须使用name配置! )
  • 组件中接收参数使用params

props配置

  • 在组件中使用 $route 会使之与其对应路由形成高度耦合，从而使组件只能在某些特定的 URL 上使用，限制了其灵活性。可以使用 props 将组件和路由解耦
  • 对象写法：如果 props 是一个对象，它会被按原样设置为组件属性。当 props 是静态的时候有用
  • 布尔模式：如果 props 被设置为true，$route.params 将会被设置为组件属性。其缺点是只能适用于params参数
  • 函数模式：你可以创建一个函数返回 props。这样你便可以将参数转换成另一种类型，将静态值与基于路由的值结合等等，其参数就是$route
  • 在路由配置中添加props配置项
  • 然后在Detail组件中接收
